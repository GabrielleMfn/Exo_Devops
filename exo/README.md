# Exos DevOps (implémentation)

## Choix d'organisation

J'ai volontairement conservé le dossier `starter-code` intact à la racine pour garder une base de référence propre, conforme au support.

Les implémentations et adaptations des exercices ont été réalisées dans ce dossier `exo/`, afin de pouvoir modifier, tester et ajouter des éléments librement sans altérer le starter d'origine.

## Correspondance des dossiers

- `00` : `00-environnement/`
- `01` : `app/`
- `02` : `.github/`
- `03` + `04` : `03-04-terraform/`
- `05` : `05-ansible/`
- `06` : `06-docker/`
- `07` : `07-kubernetes/`
- `08` : `08-helm/`

## Documentation des problèmes rencontrés

Pour chaque exercice, j'ai ajouté un `README.md` dans le dossier correspondant.

Chaque README contient :

- ce que fait l'exercice,
- les commandes pour exécuter/tester,
- les problèmes techniques rencontrés,
- les correctifs que j'ai appliqués pour résoudre ces problèmes.
